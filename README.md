# STEP1:重要的事情
devops工作职责主要分4个模块
1. 自动化，自动化是devops最基础的一块技能，把日常性的工作交给机器去执行，人负责观察和应急处理是一种良好的工作模式·
要做好自动化需要具备良好的系统管理能力，编程能力，良好的管理习惯等
2. CI/CD，关注CICD旨在关注软件的生命周期，缩短开发周期，提高开发质量是这一阶段的重要任务，所以你需要设计一个合适的
git开发流程，设计合适的管道策略，设计合适的部署策略以此来保证END TO END的稳定推进
3. 可观测性，可观测行伴随着devops的大部分工作，一个系统如何不可观测就好像生病缺乏医生一样，是一件十分有风险的事情，
为了提升可观测性，需要日志，监控，定义恰当的指标等等去提供观察，以便在必要的时候进行修复
4. 持续测试，测试往往是容易被忽略的，做出来的功能如果一段时间没有用就有可能损坏，持续的测试这些功能是有必须的，
